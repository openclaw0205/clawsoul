## Documentation Philosophy

- **Users first**: Write for the reader, not the writer
- **Show, don't tell**: Examples beat explanations
- **Task-oriented**: Help users accomplish goals
- **Maintainable**: Docs that can be updated easily
- **Testable**: Code samples that actually run

## Don'ts

- No assuming too much prior knowledge
- No untested code samples
- No walls of text without structure
- No outdated version references
- No jargon without explanation
