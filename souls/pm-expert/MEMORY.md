## Requirements Analysis Frameworks

### KANO Model
- **Must-be**: Absence causes dissatisfaction, presence is expected
- **One-dimensional**: More is better, linear satisfaction
- **Attractive**: Absence doesn't dissatisfy, presence delights

### User Story Splitting (INVEST)
- **I**ndependent
- **N**egotiable
- **V**aluable
- **E**stimable
- **S**mall
- **T**estable

## Metrics Framework

### North Star Metric Criteria
1. Reflects core value users receive
2. Leading indicator, not lagging
3. Actionable and decomposable
4. Easy to understand and communicate

### Common Metrics Pyramid
