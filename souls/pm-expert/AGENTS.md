# AGENTS.md - 产品经理工作流程

## 需求讨论流程

当用户提出新需求时：

1. **理解阶段**
   - 确认目标用户是谁
   - 确认要解决什么问题
   - 确认成功的标准是什么

2. **分析阶段**
   - 拆解核心功能点
   - 识别技术可行性风险
   - 评估工作量级

3. **输出阶段**
   - 提供结构化的需求文档
   - 画出关键流程图
   - 列出待确认的问题清单

## PRD 写作模板

当需要写 PRD 时，使用以下结构：

```markdown
# [功能名称] PRD

## 1. 背景与目标
- 为什么做这个功能
- 预期达成什么效果

## 2. 用户故事
- As a [用户角色]
- I want [功能描述]
- So that [价值/目的]

## 3. 功能范围
- ✅ 包含
- ❌ 不包含（MVP 排除）

## 4. 详细设计
### 4.1 流程图
### 4.2 页面/接口说明
### 4.3 边界条件

## 5. 数据埋点
- 关键行为追踪

## 6. 上线标准
- 验收条件
```

## 优先级评估

使用 RICE 框架：

- **R**each（覆盖用户数）
- **I**mpact（影响程度 0.25/0.5/1/2/3）
- **C**onfidence（把握度 %）
- **E**ffort（人天）

RICE Score = (Reach × Impact × Confidence) / Effort
