# MEMORY.md - Indie Hacker Knowledge

## Idea Validation Checklist

- [ ] Can I explain the problem in one sentence?
- [ ] Have I talked to 5+ potential customers?
- [ ] Are people actively looking for solutions (search volume)?
- [ ] Is someone already paying for alternatives?
- [ ] Can I reach these people affordably?
- [ ] Am I 10x better or differently positioned?
- [ ] Can I build an MVP in 2-4 weeks?

## MVP Scope Framework

### Must Have (Week 1-2)
- Core value proposition only
- Basic authentication
- Payment integration (Stripe)
- Essential UI (can be ugly)

### Should Have (Week 3-4)
- Basic onboarding
- Email notifications
- Settings page
- Documentation

### Won't Have (Post-launch)
- Team features
- Advanced analytics
- Mobile app
- API
- Everything else

## Pricing Psychology

### Anchoring
Show expensive option first to make others seem reasonable.

| Plan | Price | Purpose |
|------|-------|---------|
| Enterprise | $299/mo | Anchor |
| Pro | $49/mo | Target |
| Basic | $19/mo | Entry |

### Pricing Page Best Practices
- 3 tiers maximum
- Highlight recommended plan
- Annual discount (17-20%)
- Show monthly price, bill annually
- Feature comparison table

## Marketing Channels for Indie Hackers

| Channel | Cost | Time to Results | Best For |
|---------|------|-----------------|----------|
| SEO/Content | Low | 6-12 months | Long-term |
| Building in Public | Free | 3-6 months | Developer audience |
| Product Hunt | Free | Immediate spike | Launch boost |
| Communities | Free | 1-3 months | Niche products |
| Cold Email | Low | 1-2 months | B2B |
| Paid Ads | High | Immediate | Validated products |

## Launch Checklist

### Before Launch
- [ ] Landing page with clear value prop
- [ ] Payment working (test mode + live)
- [ ] Basic analytics (Plausible/Fathom)
- [ ] Error monitoring (Sentry)
- [ ] Backup customer support email

### Launch Day
- [ ] Product Hunt submission
- [ ] Tweet thread
- [ ] Post in relevant communities
- [ ] Email personal network
- [ ] Monitor for issues

### After Launch
- [ ] Thank early users publicly
- [ ] Collect testimonials
- [ ] Fix critical bugs immediately
- [ ] Schedule post-mortem
